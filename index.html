Untitled Game (beta)

×

This game is my implementation of a Nonogram engine.

Highlights:
  • Thick client game engine using backbone.js
  • UI layer powered by Foundation
  • Programmatic solver
  • Random (and therefore infinite) level generation

Past the issues of UI and mechanics implementation, one of the more interesting challenges has been determining whether a randomly generated board is solvable. My approach has been to build an AI solver, capable of playing the game as I do. The algorythm understands many of the game play techniques used to iterate on the game state in an attempt to solve the puzzle.

Employee Roster

×

The Employee Roster application is the web self service portal used to capture demographic, compensation, and pension enrollment data for employees of Episcopal churches and institutions. It also serves as the interface to capture physical demographic information for all kinds of institutions.

Highlights:
  • Thick client, template-driven backbone application
  • 5,000 users
  • 65,000 employee records captured
Watch the video ☞
My role:

I acted as the lead UI engineer, designing the application architecture to support a thick client model and to enable easy to scale application consistency. Backbone.js was used to provide the persistence layer and view management. jQuery UI and jQuery validation were used to provide consistent UI convenience.
